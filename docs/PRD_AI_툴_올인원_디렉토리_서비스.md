### 1.3 핵심 사용 시나리오 (최소 10개)

1. **시나리오 1 — 직장인:** “이미지 생성 AI 찾기” → 카테고리 [이미지 생성] → 가격 [무료/프리미엄]·한국어 지원 필터 → 상세 비교 → 북마크 후 나중에 보기  
2. **시나리오 2 — 마케터:** “캠페인용 카피 AI 비교” → 검색 “카피라이팅” → 정렬 [인기순] → 2~3개 툴 상세 → 비교 페이지에서 결정  
3. **시나리오 3 — 디자이너:** “이번주 트렌딩 AI” → 홈 [이번주 급상승] → 스크린샷·사용 예시 확인 → 관련 아티클(프롬프트) 읽기  
4. **시나리오 4 — 개발자:** “ChatGPT 대안” → 검색/대안 페이지 `/alternatives/chatgpt` → API·가격·플랫폼 필터 → 공식 링크 클릭  
5. **시나리오 5 — 학생:** “무료 AI만 보기” → 필터 [가격: 무료] + [학생 할인] → 상세에서 할인 링크 클릭  
6. **시나리오 6 — 재방문:** 로그인 → [내 북마크]·[내 컬렉션] → 새로 추가된 툴/업데이트 알림 확인  
7. **시나리오 7 — UGC:** 새 툴 제안 → 제출 폼 → 관리자 검수 후 반영 → 내 제출 이력·포인트 확인  
8. **시나리오 8 — 콘텐츠:** “마케터용 AI 워크플로” 아티클 읽기 → 글 내 툴 링크 클릭 → 상세·비교로 이어짐  
9. **시나리오 9 — 뉴스:** “이번주 AI 소식” → 뉴스 섹션 → 관심 툴 관련 뉴스만 필터 → 툴 상세로 연결  
10. **시나리오 10 — 공유:** “내 직무 AI 툴킷” 컬렉션 만들기 → 공유 링크 SNS 전달 → 친구가 툴 클릭아웃

### 1.5 North Star Metric 및 AARRR 퍼널

**North Star Metric (NSM)**  
- **정의:** “툴 상세 페이지에서 외부 사이트로의 **클릭아웃(Outbound Click)** 수”  
- **이유:** 발견 → 비교 → 결정 → 실제 사용(이동)까지의 핵심 전환을 측정하며, 수익(제휴/리드)과도 직결됨.

**AARRR 지표**

| 단계 | 지표 | 목표(예시) | 측정 방법 |
|------|------|------------|-----------|
| **Acquisition** | 월간 순방문자(MAU), 검색 유입 비율 | MAU 10만, 검색 40% | GA4, 검색 콘솔 |
| **Activation** | 첫 툴 클릭/상세 진입률, 첫 검색 수행률 | 60% 이상 | 이벤트: search, tool_view |
| **Retention** | 7일/30일 재방문율, 북마크 사용자 비율 | WAU/MAU 25%+ | 세션·북마크 이벤트 |
| **Referral** | 공유(컬렉션·툴)·초대·UGC 제출 수 | 공유 5% of DAU | share_click, submission |
| **Revenue** | 클릭아웃당 수익, 프리미엄 노출·제휴 수익 | (비공개 목표) | 제휴 픽셀·자체 매핑 |

## 3. UI/UX 설계 가이드

### 3.1 디자인 원칙

| 원칙 | 설명 | 구현 디테일 |
|------|------|-------------|
| **미니멀** | 한 화면에 한 가지 목적, 불필요한 배너·팝업 최소화 | 히어로+카드 그리드 위주, CTA 1~2개 |
| **카드 기반** | 툴·글·뉴스 단위를 카드로 통일, 스캔 가능 | 카드: 썸네일·제목·1줄 요약·메타(가격/한국어)·호버 시 액션 |
| **충분한 여백** | 가독성·집중도 향상 | 그리드 갭 24px+, 카드 패딩 16~24px, 섹션 간 48px+ |
| **타이포 스케일** | 명확한 위계 | H1 24~32px, H2 20~24px, 본문 16px, 캡션 14px, 일관된 line-height |
| **다크모드** | 선호 사용자·접근성 | 시스템/수동 전환, 대비 유지(WCAG AA), 색상 토큰 공통 |

### 3.2 홈 화면 구성안 (2~3가지)

**안 A — 검색 중심**  
- 상단: 히어로 + 큰 검색창 + 인기 검색어/자동완성  
- 중단: “이번주 급상승” 6~8개 카드 → “오늘의 추천” 6개  
- 하단: 카테고리 그리드(아이콘+이름) → 최신 뉴스 3줄 + “글 더보기” CTA  

**안 B — 큐레이션 중심**  
- 상단: 짧은 히어로 + 검색  
- 중단: “직무별 베스트” 탭(마케터/디자이너/개발자/스타터) → 각 4~6개 카드  
- 하단: 트렌딩 리스트 + 카테고리 그리드 + 뉴스/아티클 프리뷰  

**안 C — 콘텐츠 균형**  
- 상단: 히어로 + 검색 + “무료로 시작하기” CTA  
- 중단: 추천 / 트렌딩 / 랭킹 탭으로 전환 가능한 한 줄 카드  
- 하단: “이번주 AI 소식” 3개 + “실무자가 쓴 글” 3개 + 카테고리  

**결정:** MVP는 안 A, V1에서 안 B 요소(직무별) 추가. A/B 테스트로 CTA·전환율 비교.

| 가격 배지 | 권장 | 무료/유료/프리미엄/무료+유료 |
| 한국어 지원 | 권장 | Y/N 배지 |
| 카테고리(1~2개) | 필수 | 링크 |
| 태그(2~4개) | 선택 | 클릭 시 검색 |
| 북마크 버튼 | V1 | 로그인 시 |
| “유사 툴” 툴팁 | V1 | 호버 시 2~3개 이름 |

### 3.5 상세 페이지 레이아웃

- **상단:** 로고·이름·한 줄 요약·평점(있을 경우)·CTA(공식 사이트·북마크·비교 추가)  
- **미디어:** 스크린샷/영상 갤러리(캐러셀)  
- **핵심 기능:** 불릿 리스트  
- **사용 예시:** 1~2개 시나리오 또는 프롬프트 예시  
- **장단점:** 사용자 리뷰 요약 또는 편집 큐레이션  
- **가격표:** 플랜별 요약, “최신 업데이트: YYYY-MM-DD”  
- **대안 비교:** “비슷한 툴” 카드 4~6개, “A vs B 비교” 링크  
- **FAQ:** 아코디언 3~5개  
- **관련 글/뉴스:** 같은 툴 태그 아티클·뉴스 카드

## 4. 데이터 모델 / DB 스키마 / 수집-검수 파이프라인

### 4.1 엔티티 목록

- **Tool** — 툴 마스터(이름, slug, URL, 설명, 로고, 카테고리, 태그, 가격 유형, 한국어 지원, 플랫폼, 신뢰도 스코어, 상태)

- **Tag** — 태그(이름, slug, 사용 횟수)  
- **PricingPlan** — 툴별 가격 플랜(툴 FK, 플랜명, 가격, 통화, 주기)  
- **Feature** — 툴 기능(툴 FK, 기능명, 설명)  
- **LanguageSupport** — 툴별 지원 언어(툴 FK, 언어 코드)  
- **Platform** — 웹/앱/API/브라우저 확장 등  
- **Article** — 글(제목, slug, 본문, 작성자, 툴 연관, 발행일)  
- **News** — 뉴스(제목, URL, 출처, 일자, 관련 툴)  
- **User** — 사용자(이메일, 소셜, 역할)  
- **Bookmark** — 북마크(유저, 툴)  
- **Review** — 리뷰(유저, 툴, 점수, 내용, 상태)  
- **Vote** — 추천/비추(유저, 대상: 툴/리뷰, up/down)  
- **Collection** — 컬렉션(유저, 제목, 설명, 툴 목록, 공개 여부)  
- **Submission** — 제출 제안(툴 신규/수정, 제출자, 상태, 검수 메모)  
- **ChangeLog** — 툴 변경 이력(툴, 변경 유형, 필드, 이전/이후 값, 일시)  

---

### 4.2 수집-검수 파이프라인

1. **입력:** 크롤링(정기)·수동 제보(UGC)·파트너 제휴(API/피드)  
2. **중복 감지:** URL 정규화 + 이름 유사도(Levenshtein) → 후보 매칭 → 관리자 확인  
3. **검수:** Submission 테이블에 “대기” → 관리자 승인/반려 → 반영 시 Tool/ChangeLog 갱신  
4. **노출:** 변경 이력은 상세 페이지 “최근 변경” 섹션에 표시  

---

### 4.3 최신성·품질 정책

- **최신성:** 크롤링 주기(예: 가격 페이지 2주), UGC 즉시 검수 큐 반영, “마지막 검증일” 상세 페이지 노출  
- **스팸/광고/낚시 방지:** URL 도메인 화이트리스트 검사, 제출 빈도 제한, 신뢰도 스코어(리뷰·투표·출처 가중)로 정렬 반영  
- **신뢰도 스코어:** (가중 합) 리뷰 수·평균 평점·투표·클릭아웃·관리자 검증 여부 → 랭킹/트렌딩에 반영

## 5. 추천 / 랭킹 / 트렌딩 로직

### 5.1 트렌딩 점수 산식 (제안)

```
trending_score = w1·views_7d + w2·bookmarks_7d + w3·clickouts_7d
                 + w4·review_count_7d + w5·(growth_rate_views)
                 + w6·return_visitor_contribution
```

- **가중치 예:** w1=1, w2=3, w3=2, w4=2, w5=1.5, w6=1 (조정 가능)  
- **growth_rate:** 최근 7일 / 이전 7일 비율, 상한 적용  
- **return_visitor_contribution:** 재방문 사용자가 클릭/북마크한 비율  

**결정:** 주간 단위 집계, 캐시(Redis) 1시간 갱신, “이번주 급상승” 노출.

---

### 5.2 개인화 추천 (V1~V2)

- **관심 카테고리:** 최근 조회/북마크 카테고리 기반 툴 추천  
- **직무 선택:** 온보딩 또는 설정에서 “마케터/디자이너/개발자/스타터” 선택 → 해당 큐레이션 리스트  

---

### 5.3 큐레이션 슬롯

| 슬롯 | 설명 | 데이터 소스 |
|------|------|-------------|
| 오늘의 AI 툴 | 일 1개 편집자 픽 또는 트렌딩 1위 | 수동 또는 trending_score 1위 |
| 이번주 급상승 | 트렌딩 점수 상위 10개 | trending_score |
| 업무별 베스트 | 직무별 상위 N개 | 카테고리+평점+클릭아웃 조합, 또는 수동 큐레이션 |
